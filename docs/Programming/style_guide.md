# Python Style Guide

[Download the Python style guide](../assets/documents/python_style_guide.txt){:download="python_style_guide.txt"}

This page points to the canonical Python style guide, which can be downloaded and used directly in projects or with AI-assisted workflows.

The guide defines expectations for naming, comments, formatting, imports, type hints, docstrings, and package build structure. It also includes the preferred minimal `pyproject.toml` approach for Python modules using Hatch with versions derived from Git tags.

It is not intended as a rigid rulebook. It documents the style I prefer and use across my projects to establish consistency.
