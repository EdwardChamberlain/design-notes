# Engineering Notes

## Purpose
Create a reliable engineering notebook with reusable rules, assumptions, and quick worked examples.

## Key rules
- Capture assumptions before calculations.
- Prefer SI units and include conversions explicitly.
- Link every claim to a standard, datasheet, or test note.

## Worked example
Use this starter map to place new notes consistently.

| Domain | Start page | Typical outputs |
| --- | --- | --- |
| Threads | `threads/` | Callouts, strength checks |
| GD&T | `gdt/` | Datum schemes, tolerance stacks |
| DFM | `dfm/` | Process constraints, cost drivers |
